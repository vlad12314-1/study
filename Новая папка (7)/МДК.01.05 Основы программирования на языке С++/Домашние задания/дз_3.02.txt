Алгоритмы STL (<algorithm>)
1. Что такое алгоритмы STL
Алгоритмы STL — это готовые функции, которые:

Работают с диапазонами элементов

Не знают, с каким конкретным контейнером работают

Используют итераторы для доступа к элементам

Не хранят данные, только обрабатывают их

2. Преимущества использования <algorithm>
Упрощают код (рисунки 1-2 демонстрируют разницу)

Снижают вероятность ошибок

Повышают читаемость кода

3. Диапазоны: begin() и end()
Почти все алгоритмы принимают два итератора:

cpp
algorithm_name(begin_iterator, end_iterator, ...)
Важно:

begin() — указывает на первый элемент

end() — указывает на позицию после последнего элемента

4. Основные алгоритмы
sort() — сортировка элементов
cpp
std::sort(v.begin(), v.end());
find() — поиск элемента
cpp
auto it = std::find(v.begin(), v.end(), value);
if (it != v.end()) { /* найден */ }
// Если не найден: it == v.end()
count() и count_if()
count() — считает количество конкретных значений

count_if() — считает элементы по условию (предикату)

find_if() — поиск первого элемента по условию
for_each() — применение операции к каждому элементу
Не изменяет контейнер (если явно не указано)

Используется для побочных эффектов (вывод, логирование)

remove() и remove_if()
Важно: remove() не удаляет элементы физически!

Сдвигает "ненужные" элементы в конец контейнера

Возвращает итератор на новый логический конец

Для фактического удаления используется erase-remove идиома:

cpp
v.erase(std::remove(v.begin(), v.end(), value), v.end());
all_of / any_of / none_of
all_of — все элементы удовлетворяют условию

any_of — хотя бы один элемент удовлетворяет условию

none_of — ни один элемент не удовлетворяет условию

min_element() / max_element()
Возвращают итератор на минимальный/максимальный элемент

Не возвращают само значение напрямую

transform() — применяет функцию к каждому элементу
cpp
std::transform(v.begin(), v.end(), v.begin(), функция);
Может изменять контейнер на месте или копировать в другой

copy() — копирует диапазон элементов
cpp
std::copy(source_begin, source_end, destination_begin);
Важно: Назначение должно иметь достаточный размер

5. Алгоритмы и строки
std::string — это тоже контейнер, поэтому с ним работают все алгоритмы STL

6. Почему алгоритмы лучше циклов
Преимущества алгоритмов:

Короче и читаемее

Меньше вероятность ошибок

Оптимизированы компилятором

Стандартизированы

Циклы:

Иногда необходимы

Но не всегда являются лучшим выбором

Ключевые выводы:
Алгоритмы STL работают через итераторы, абстрагируясь от конкретных контейнеров

Для полного удаления элементов используйте erase-remove идиому

Всегда проверяйте результат find() сравнением с end()

Алгоритмы повышают качество кода: делают его короче, понятнее и надежнее
